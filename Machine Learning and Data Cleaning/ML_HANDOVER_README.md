# 🧠 Machine Learning Engine: Handover & Execution Guide

This guide provides full instructions for a backend developer to run, integrate, and maintain the PathFinder+ Recommendation Engine.

---

## 🛠️ 1. Prerequisites & Environment Setup

The engine is built with **Python 3.10+** and uses **PyTorch** for embeddings.

### **Step 1: Install Dependencies**
Run this in the root `Machine Learning and Data Cleaning/` directory:
```bash
pip install -r requirements.txt
```

### **Step 2: Configure Environment**
Create a `.env` file in the root directory (copy from existing or create new):
```env
MONGO_URI="your_mongodb_atlas_connection_string"
DATABASE_NAME="pathfinder_plus"
# Optional: Only if using the Chatbot feature
GEMINI_API_KEY="your_google_ai_key"
```

---

## 🚀 2. Running the Engine (Initialization)

The engine is encapsulated in the `RecommendationEngine` class. For production, use the cloud-native factory method which pulls data directly from MongoDB Atlas.

### **Quick Start Code**
```python
from core.recommendation_engine import RecommendationEngine

# 1. Initialize from Cloud (Production Mode)
engine = RecommendationEngine.from_mongo()

# 2. Prepare Assessment Vector
# This vector is typically generated by 'process_comprehensive_assessment'
assessment_vector = {
    "status_level": 2,          # 0: O/L, 1: A/L, 2: Undergrad, 3: Pro
    "experience_years": 3,
    "responsibility_band": 2,   # 0-4 scale
    "education_level": 3,       # 1-6 scale
    "extracted_intent_skills": ["Python", "SQL", "Excel"],
    "domain": "IT"
}

# 3. Get the definitive 11-Point Dashboard Bundle
bundle = engine.get_recommendations_from_assessment(
    assessment_vector=assessment_vector,
    target_job="Data Analyst"
)

# 4. Access the results
print(f"Readiness: {bundle['readiness_score']['overall']}%")
print(f"Top Course: {bundle['recommendations'][0]['course_title']}")
print(f"Top Mentor: {bundle['mentors'][0]['name']}")
```

## 💾 3. Managing ML Model Artifacts

The engine uses semantic search powered by **SBERT**. The "Models" are stored as `.pt` (PyTorch) files in the `models/` folder.

### **Must-Have Files in `models/`:**
*   `job_embeddings.pt`: Vector index for 3,000+ jobs.
*   `course_embeddings_all_courses_master.pt`: Vector index for professional courses.
*   `esco_occ_embeddings.pt`: Mapping index for 2,500+ standard occupations.

> [!IMPORTANT]
> **If you add new data to MongoDB**, you MUST regenerate these embeddings so the engine can search them. Run:
> ```bash
> python scripts/generate_model_artifacts.py
> ```

## 🛰️ 4. Data Collection & Pipeline

The system uses an automated pipeline to keep the Cloud database fresh.

| Component | Responsibility |
| :--- | :--- |
| **Orchestrator** | `scripts/orchestrate_scraping.py` — Runs the full sequence below. |
| **Scrapers** | Located in `scrapers/` (Ikman, TopJobs, Xpress, PickACourse). |
| **Merge & Clean** | Normalizes job/course titles and removes duplicates. |
| **Embedding Generation** | Rebuilds `.pt` files for semantic search. |
| **Mongo Sync** | `scripts/push_to_mongo.py` — Pushes local CSVs to Atlas. |

**To run the full ingestion pipeline:**
```bash
python scripts/orchestrate_scraping.py
```

## 🤖 6. Hybrid ML Layer

Beyond SBERT, the engine supports a **Hybrid ML Layer** (Random Forest + Gradient Boosting) for high-accuracy classification.
*   **Location**: `core/ml_classifier.py`
*   **Training**: Automatically trains on first load if models are missing in `models/`.
*   **Usage**: Augments semantic results with statistical classification signals.

---

## 🔗 7. Integration Notes
*   **Input**: The engine primarily expects an `assessment_vector` (dictionary with skills, level, and domain) and a `target_job` (string).
*   **Output**: Returns a JSON-compatible dictionary containing `recommendations`, `jobs`, `mentors`, `readiness_score`, and a step-by-step `action_plan`.
*   **Performance**: The first initialization takes 3-5 seconds (loading models into memory). Subsequent calls are sub-second (<200ms).
